# Crawler Audit & Refinement Plan — AEC Vendor Target List

**Date:** May 8, 2026
**Source list:** `companies.csv` (40 exhibitors/sponsors from an AEC tech event)
**Pitch shape:** Arvaya AI Consulting → **AEC software vendors** (not AEC firms)

---

## 1. The big picture

Your existing crawler is tuned for two shapes of prospect:

1. **AEC firms** — general contractors, design firms, owners. The `researcher.md` prompt has an explicit branch for these (SAM.gov, USASpending, Procore/Autodesk/Bluebeam stack). This is what the Brantley Construction run hit.
2. **General companies** — the breadth-then-deep Perplexity loop.

Your 40 companies are **neither**. They're software vendors *selling into* AEC. The pitch Arvaya leadership is making is "let us help you ship AI features in your product, train your team, or modernize the data plane behind your platform." That's a different brief. The intel that matters is:

- **Their AI roadmap** (do they already ship AI features? Where are the gaps?)
- **Their ML/data hiring posture** (open roles, glassdoor signals)
- **Their tech stack and engineering org** (GitHub, engineering blog, conference talks)
- **Their funding/ownership** (PE-backed? Recently funded? Bootstrapped?)
- **Their AEC customer case studies** (proof they actually serve this market)

The SAM.gov branch is irrelevant for ~all 40 of these.

### 3.2 Vendor-shape data the current prompt won't fetch

| Signal | Current prompt covers it? | What to add |
|---|---|---|
| GitHub org / repo activity | No | New `web_search` query: `site:github.com "<Company>"` filtered to last 12 months |

## 4. Recommended crawler changes

In priority order. None of these are big lifts.

### Change 1 — Add name-anchoring to every Perplexity query (highest ROI)

**File:** `src/prompts/researcher.md`

Add a new "Phase -1 — Disambiguation" section before Phase 0:

> Before any Perplexity call, build an **anchor block** from the lead:
> - The full legal company name AND any rebrand history if known
> - The website domain (strip protocol)
> - One product or service phrase from the lead (productFocus)
>
> Every Perplexity query MUST embed this anchor. Example:
> ```
> Research "Box Inc" (box.com, cloud content management for AEC).
> What is the latest news, executive interviews, funding, hiring activity...
> ```
>
> Drop any source URL whose domain or page title doesn't reference the anchor company. Cite the rejection in your reasoning if a marginal call.

### Change 2 — Add a `prospectArchetype` field to Lead

**Files:** `src/types.ts`, `src/index.ts`, all lead JSONs

Extend `LeadSchema`:

```ts
prospectArchetype: z.enum(["aec_firm", "aec_vendor", "other"]).optional()
```

Branch in `researcher.md`:

- `aec_firm` → existing SAM.gov / Procore-stack branch.
- `aec_vendor` → new branch (see Change 3).
- `other` → general-purpose flow.

This avoids the AEC-firm SAM.gov branch firing on Egnyte. Cost saving alone justifies the change.

### Change 3 — Add an "AEC vendor" branch in researcher.md

When `prospectArchetype === "aec_vendor"`, run instead:

```
1. perplexity_search:
   "<Company>" AEC architecture engineering construction customers case studies — recent product or AI feature releases

2. perplexity_discovery focus=["job_postings"], targetDepartment="Machine Learning":
   recent ML, AI, or data engineering job postings — what tech stack, which models, hiring scale

3. perplexity_search:
   site:github.com "<Company>" OR org pages — public repos, recent activity

4. perplexity_discovery focus=["financials"]:
   funding history, ownership, PE/VC backing, recent acquisitions or product launches

5. perplexity_discovery focus=["people"]:
   founders, CEO, CTO, Head of Engineering, Head of AI/ML, Head of Product
```

Skip the SAM.gov / Procore-stack branch entirely.

### Change 4 — Soften the 8-source minimum for thin-footprint targets

In `researcher.md`:

> If the breadth scan returns fewer than 10 citations total across all `perplexity_search` calls, the **minimum sources** drop to **4** (not 8). Note the reason in a special source `id="src-meta-thin"` with `category="other"` and snippet `"Footprint thin: only N citations available across all phases."`

This will change `passedVerification` thresholds — verify the QAVerifier is OK with 4-source briefs (probably needs a parallel softening in `qaVerifier.md`).

### Change 5 — Add `excludeKeywords` to Lead

**Files:** `src/types.ts`, `src/prompts/researcher.md`

```ts
excludeKeywords: z.array(z.string()).optional()
```

Researcher must drop any source URL whose page title or snippet contains any string in `excludeKeywords`. For Brantley Construction we'd set `["Brantley County wildfire", "Brantley County, GA"]`. For Box, `["box office", "cardboard"]`. For Joist AI, `["joist app", "consumer contractor"]`.

### Change 6 — Update the docs

Independent of the rest. README and `docs/build-summary.md` both describe an Anthropic + Apollo + web_search system that no longer exists. Anyone reading them today gets a wrong picture. Two specific fixes:

- README header should say OpenAI gpt-4o, Perplexity sonar-deep-research + sonar-pro, Firecrawl. The "Migration from Apollo" section can be deleted.
- `build-summary.md` decision log needs a new entry "decision #15: Switched from Anthropic + Apollo back to OpenAI + Perplexity" with the date and reasoning.

I'll do this in a follow-up if you say yes.

## 5. Cost-aware test plan

Running all 40 companies as-is, with no changes: **~$22–$48** ($0.55–$1.20/run × 40). Several of those runs will be wasted on name-collision junk and need re-runs.

### Phase 1 — Validation set (3 runs, ~$2–4)

Pick three companies that span the failure modes:

| # | Company | Tests what |
|---|---|---|
| 1 | **Egnyte** | Tier A control. Should produce a strong brief out of the box. |
| 2 | **Joist AI** | Tier B + name-collision (Joist consumer app). Tests Change 1 (name-anchoring). |
| 3 | **Seev** | Tier C, thin footprint, generic name. Tests Change 4 (relaxed minimum). |

Run all three with the **current** crawler first to establish a baseline. Then apply Changes 1, 2, 3 (the cheap ones) and re-run. Compare side by side. ~$4–8 total at this stage.

### Phase 2 — Tiered rollout (~$15–25)

If Phase 1 looks good:

- Run all 7 Tier A companies first. They should produce the cleanest briefs and earliest leadership wins.
- Run all of Tier B next, with name-anchor and exclude-keywords applied per-company where flagged in §3.1.
- Run Tier C last, with relaxed minimums; expect 30–50% to come back marginal.

### Phase 3 — Reconcile

After Phase 2, look at runs that came back with `passedVerification: false` and decide per-company whether to re-run with extra context, manually augment the brief, or drop the company from the active pursuit list.
